# 메트릭 및 판단 기준

## 수집 메트릭

### 성능 (k6)

| 메트릭 | 설명 | 기준 |
|--------|------|------|
| `http_req_duration` p50/p95/p99 | 응답시간 | 낮을수록 좋음 |
| `http_reqs` rate | 초당 처리량 (RPS) | 높을수록 좋음 |
| `http_req_failed` | 에러율 | 0%에 가까워야 |
| `http_req_waiting` | TTFB | BFF 내부 처리 시간 |

### JVM (Micrometer / JFR)

| 메트릭 | 설명 | 기준 |
|--------|------|------|
| `jvm.threads.live` / `peak` | 활성/피크 스레드 수 | VT 도입 시 감소 예상 |
| `jvm.gc.pause` | GC 횟수/시간 | 총 pause 비교 |
| `jvm.memory.used` heap/non-heap | 메모리 사용량 | VT 경량 스택으로 감소 |

### 컨테이너 (cAdvisor / Prometheus)

| 메트릭 | 설명 |
|--------|------|
| `container_cpu_usage_seconds_total` | Pod CPU |
| `container_memory_working_set_bytes` | Pod 메모리 (RSS) |

## JFR 녹화 방법

```bash
# JVM 시작 옵션
java -XX:StartFlightRecording=duration=300s,filename=test.jfr -jar bff.jar

# 실행 중 녹화
jcmd <PID> JFR.start name=perf-test duration=300s filename=/tmp/test.jfr
```

## k6 → Prometheus 연동

```bash
k6 run --out experimental-prometheus-rw \
  --env K6_PROMETHEUS_RW_SERVER_URL=http://prometheus:9090/api/v1/write \
  scenario.js
```

## 판단 기준

> Virtual Thread 도입이 유의미하다고 판단하는 기준:

1. **고동시성(VU 500+)에서 p95 latency 30% 이상 감소**
2. **Peak thread 수 10배 이상 감소** (platform thread 기준)
3. **동일 부하 대비 CPU 동등 또는 감소**
4. **에러율 차이 없음** (기능 회귀 없음)
